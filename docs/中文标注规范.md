# 中文标注规范

这份规范用来保证仓库里的说明、脚本输出和示例图纸都能被中文用户直接看懂。新增内容时优先按这里的规则检查一遍。

## 一、每个区域必须有中文名称

仓库里的主要目录、文档段落和脚本输出都应该说明自己属于哪个区域：

| 区域名称 | 范围 | 必须说清楚 |
| --- | --- | --- |
| 总览区 | `README.md`、`docs/中文介绍.md` | 项目做什么、不做什么、先看哪里 |
| 状态检查区 | `examples/bridge_status.py` | 每条桥是否可用、缺什么、下一步怎么处理 |
| 图像生成区 | `examples/comfy_bridge/` | ComfyUI 地址、checkpoint、队列、输出路径 |
| 工程制图区 | `cad-mcp-autocad/`、`scripts/` | 图层、孔位、中心线、尺寸、区域名称 |
| 修图抠图区 | `examples/photoshop_bridge/` | 输入图、输出图、Photoshop COM、主体选择 |
| 三维扩展区 | `docs/04-codex-blender.md` | Blender 可执行文件、MCP 桥、公开安全示例 |
| 安全隔离区 | `.gitignore`、`AGENTS.md` | 不提交的账号、模型、素材、输出和缓存 |

## 二、每个表格必须有中文列名

推荐列名：

- `区域`
- `文件或目录`
- `中文用途`
- `运行命令`
- `本机配置`
- `安全边界`
- `处理建议`

只有命令、路径、API、MCP、workflow、prompt、checkpoint 这类技术词保留英文。

## 三、每个脚本输出必须有中文标签

命令行输出不要只打印 `output=`、`count=` 这类英文键。推荐写成：

```text
输出文件：<local-output>\demo.dwg
图元数量：120
处理建议：先启动 ComfyUI 后重试
```

如果需要 JSON，保留英文 key 方便机器读取，同时增加 `label` 或 `status_label` 这类中文说明字段。

## 四、每张示例 CAD 图必须有中文区域标注

CAD 示例图纸至少标出：

- 图纸标题，例如“连接板参数化示例”。
- 主体区域，例如“主圆孔区”“左上安装孔区”“右下小孔区”。
- 功能区域，例如“外轮廓连接区”“中心线基准”“尺寸标注区”。
- 安全说明，例如“示例图纸，仅用于公开演示，不含客户数据”。

尺寸符号可以保留 `%%c44`、`R80`、`69%%d` 等 CAD 标准写法，但旁边应有中文区域名称。

## 五、每个 README 必须先讲人话

README 的推荐顺序：

1. 这个目录做什么。
2. 这个目录不放什么。
3. 文件中文用途表。
4. 运行前准备。
5. 运行命令。
6. 参数中文说明。
7. 安全边界。

## 六、提交前检查

提交前至少运行：

```powershell
python -m unittest discover -s tests
python -m compileall examples cad-mcp-autocad/src scripts tests
```

如果改了文档，还要快速检查 `README.md`、`docs/中文用途索引.md` 和本规范是否互相链接。
